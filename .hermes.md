# Project Context — Venezuela Te Ayuda

- Type: Next.js 16 help/mutual-aid map web app
- Stack: Next.js 16 App Router, React 19, TypeScript 5, Tailwind CSS 4, shadcn/ui
- DB: Postgres via Prisma 6 + Prisma Migrate
- Auth: NextAuth v5 beta, email verification, roles AYUDANTE/SOLICITANTE/ADMIN
- Maps: Leaflet + react-leaflet + OpenStreetMap
- E2E: Playwright

## Commands
- `npm install` — dependencies
- `npx prisma migrate dev` — local migrations
- `npm run seed` — seed admin user
- `npm run dev` — dev server
- `npx playwright test` — e2e tests
- `npm run vercel-build` — `prisma generate && next build`

## Layout
- `src/app/` — App Router routes, including `/mapa`, `/admin`, `/legal/...`
- `src/components/` — UI components, map, moderation, admin panels
- `prisma/` — schema + migrations + seed
- `src/lib/email.ts` — email provider; swap provider for production

## Conventions
- Spanish UI copy
- All users and cards default to PENDIENTE until admin approval
- Chat is internal only; do not expose phone/email
- Playwright tests target dev server on port 3100
- Moderation audit trail required; do not remove audit entries

## Don'ts
- Do not skip email verification in auth flow
- Do not expose contact info outside internal chat
- Do not approve users/cards without admin action
- Do not remove audit logs
